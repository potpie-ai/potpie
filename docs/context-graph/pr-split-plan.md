# PR-split plan — `feat/graph-updates` → `main`

> Generated by a deep-dive workflow (9 slice agents + synthesis) over 442 files / ~46k insertions / ~16k deletions across 22 commits. Commit boundaries do **not** map to these PRs — see Reconstruction strategy.

## Overview

feat/graph-updates is one large pivot — Potpie moves OFF deterministic scanning / connector ingestion (Jira/Linear/scanners/belief/sync) and ONTO a harness-authored Graph V1.5/V2 "workbench" (catalog/read/search-entities/mutate/propose/commit/inbox/quality/nudge), backed by a FalkorDB-lite + embedding-aware claim store, served by a restored detached daemon with a read-only React graph-explorer UI. The 442-file / ~46k-insert diff decomposes into a clean dependency tree: shared DOMAIN primitives land first (graph_contract + ontology), then the read surface, write/mutation contract, V2 envelope, and persistence DTOs; the APPLICATION/ADAPTER half (embedder+claim-query, FalkorDB, semantic-mutation pipeline, GraphWorkbenchService, readers, nudge) builds on those; the CLI workbench, daemon lifecycle, and UI sit on top; the legacy scanner/connector/ingest subsystems are removed last (after every consumer is cut over); and a handful of genuinely independent fixes/deps/docs ride alone. The two squash-y commits (graph-workbench mega-commit + 6d4647e7 "repo ingestion + agents.md") mix concerns and must be carved by PATH/HUNK, not by commit. Tests and docs are NOT a bucket — each test file attaches to the feature/removal PR it pins; only forward-looking design docs stand alone.

General mechanic: branch each PR off its dependency (a stack), then `git checkout main -- <paths>` (or `git restore --source=feat/graph-updates -- <paths>` onto a main-based branch) to stage exactly that PR's file set, commit, and open against the prior PR in the stack. Because the slices already partitioned by path and confirmed "no remaining importers" for each removal, path-based staging is reliable for whole-file adds/deletes (the bulk of the diff).

Three hunk-level splits cannot be done by path and need `git add -p` / manual patch surgery on a single file:

2. auth_commands.py and github_commands.py are split between PR-25 (Abort/countdown/_run_tracked guards) and PR-26 (capture_usage_* call sites). Stage PR-25's hunks first, then PR-26 adds only the analytics lines.
